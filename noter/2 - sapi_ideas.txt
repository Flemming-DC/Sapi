functions (and other runtime references):
	function names must be stored in a special system table (called runtime_functions) 
		and then referenced by other tables.
	stores function using name and path to script or using dot-notation
	crate table tab (func bigint references runtime_functions, var text)
	sapi.read("select func, var from tree").parse_runtime_references().rows()
	if possible, omit the call to parse_runtime_references. 
	other runtime references might include scripts (for access control) and types (possibly useful for polymorphism)
	
sapi server:
	sapi server is a scheduled job thats runs 24-7 and gets restarted up to X times if not running. 
	server_controller is another scheduled job, which can start and stop the server. 
	Editor displays an error if server or server_controller aren't not running when expected. 
	Editor contains a refesh server button, which will stop the server copy-paste the python 
		code of the current projects root directory into the servers root directory and then start the server.
	refesh server also exists as a functions call. 
	server should check the command table very frequently and should start jobs in parrallel to minimize wait time.
		
	sapi.server.call(function)
		this assumes that function is a runtime ref 
		calls a sql procedure, that inserts a line into a command table, if (current_user, function) is inside
		a table of privileges. If not, then raise a permission denied. Arguments go into a json column in the 
		commands table.
	
	if possible, allow server_call(runtime_function) in sapi code. This can be used in check constraints
