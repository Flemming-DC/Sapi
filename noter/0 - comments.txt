
get select statements:
    split by ;
    track bracket count
    track select token
    get line and bracket_count of select token (line_start, bc_start) and continue until 
        bracket_count drops below bc_start. then get line (line_end), which is the end of the query.

for each select statement: find from token and continue parsing until hitting another clause or hitting the end of string

(1) A graph is a tree with N nodes iff it has N - 1 edges directed from parent to child, such that every child has exactly 1 parent.
(2) If you add a node with one edge and one parent to a tree, then it remains a tree.
(3) given a undirected graph and a putative root, you direct the edges from parent to child iteratively starting at the root 
    and going down to the leaves. Then (1) and (2) can be used to determine if it is and reamins a tree
point 1, 2, 3 above can be used by sqlapi to confirm and putative trees really are tree or to extend trees automatically.
e.g. this would allow one to de myTree = Tree(rootTable).extend()
